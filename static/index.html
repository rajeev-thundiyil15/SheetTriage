CSV Format

Your file needs a header row. Column names must match the keys in your schema exactly (case-sensitive). The agent handles mixed types, bad formatting, and messy data.

Example dirty input:

id,name,email,age,salary,department,start_date 1,Alice,alice@example.com,32,75000,Engineering,2021-03-15 2,Bob,bob-no-at-sign,25,sixty thousand,Sales,2020-07-01 3,Carol,carol@example.com,15,52000,Marketing,2022-01-10 4,,dave@example.com,40,95000,HR,2019-05-20 5,Eve,eve@example.com,29,-500,Finance,2023-02-28 6,Frank,frank@example.com,38,88000,Enginneering,2018-11-03

The agent catches: bad email, salary as text, age below minimum, missing name, negative salary, and the misspelled department.

Schema JSON Format

Each key under "columns" must match a column header in your CSV. Only defined columns are validated — extras are ignored.

// Full example with all options { "name": "employees", "columns": { "id": { "dtype": "int", "nullable": false }, "email": { "dtype": "str", "nullable": false, "pattern": ".+@.+\\..+" }, "age": { "dtype": "int", "nullable": false, "min_value": 18, "max_value": 100 }, "salary": { "dtype": "float", "nullable": false, "min_value": 0 }, "department": { "dtype": "str", "nullable": false, "allowed_values": ["Engineering", "Sales"] }, "start_date": { "dtype": "date", "nullable": false } } }

Schema Reference

FieldTypeDescriptionExample
dtype required Expected data type for the column "int", "float", "str", "date"
nullable boolean Whether empty values are allowed. false = required field false
min_value number Minimum allowed value (inclusive) 0, 18
max_value number Maximum allowed value (inclusive) 100, 5
allowed_values array Whitelist of valid values. Close matches (typos, casing) are fixed automatically. ["Yes", "No"]
pattern regex Regular expression the value must fully match ".+@.+\\..+"

Tips

  • Column names in the CSV must exactly match the schema keys (case-sensitive).
  • You only need to include columns you want validated — extras are ignored.
  • For allowed_values, the agent corrects close matches before marking a row as an error.
  • In Clean + Report mode, no rows are deleted — errors are annotated with an _errors column.
  • Start simple: define just dtype and nullable first, then layer in constraints.
