🚀 Quick Start

1
Prepare your CSV file — any spreadsheet exported as CSV works.
2
Write a schema JSON that describes the rules each column must follow.
3
Upload the CSV, paste (or edit) the schema in the Agent tab, click Clean Data.
4
Watch the agent inspect, find violations, and fix them in real time. Download the clean file when done.

📄 CSV Format

Your CSV needs a header row. Column names in the CSV must exactly match the keys in your schema. Everything else is flexible — the agent handles mixed types, bad formatting, and messy data.

Example CSV (dirty):

id,name,email,age,salary,department,start_date 1,Alice,alice@example.com,32,75000,Engineering,2021-03-15 2,Bob,bob-no-at-sign,25,sixty thousand,Sales,2020-07-01 3,Carol,carol@example.com,15,52000,Marketing,2022-01-10 4,,dave@example.com,40,95000,HR,2019-05-20 5,Eve,eve@example.com,29,-500,Finance,2023-02-28 6,Frank,frank@example.com,38,88000,Enginneering,2018-11-03

The agent will catch: bad email format, salary as text, age below minimum, missing name, negative salary, and the misspelled department.

📋 Schema JSON Format

The schema tells the agent what valid data looks like for each column. Every key under "columns" must match a column header in your CSV.

Full example with all options:

// schema/employees.json { "name": "employees", // any label — used in logs "columns": { "id": { "dtype": "int", "nullable": false }, "name": { "dtype": "str", "nullable": false }, "email": { "dtype": "str", "nullable": false, "pattern": ".+@.+\\..+" // regex — must contain @ and a dot }, "age": { "dtype": "int", "nullable": false, "min_value": 18, "max_value": 100 }, "salary": { "dtype": "float", "nullable": false, "min_value": 0 // no negative salaries }, "department": { "dtype": "str", "nullable": false, "allowed_values": ["Engineering", "Sales", "Marketing", "HR", "Finance"] }, "start_date": { "dtype": "date", "nullable": false } } }

⚙️ Schema Rules Reference

Rule Type What it does Example
dtype required The expected data type for the column "int", "float", "str", "date"
nullable boolean Whether empty/missing values are allowed. false = required field false
min_value number Minimum allowed numeric value (inclusive) 0, 18
max_value number Maximum allowed numeric value (inclusive) 100, 999999
allowed_values array The only values this column may contain. Agent fixes typos and casing automatically. ["Yes", "No"]
pattern regex string A regular expression the value must fully match ".+@.+\\..+" for email

🔧 What the Agent Fixes

  • Wrong type — e.g. salary stored as text ("sixty thousand") → converts or drops
  • Out-of-range values — e.g. age = 15 (below min 18) → drops the row
  • Missing required values — e.g. empty name field → drops the row
  • Invalid email / pattern — e.g. "bob-no-at-sign" → drops the row
  • Disallowed values — e.g. "Enginneering" → corrected to "Engineering"
  • Bad date formats — e.g. "15/06/2017" → converts to standard date or drops

💡 Tips

  • Column names in the CSV must exactly match keys in your schema (case-sensitive).
  • You only need to include columns you want validated — extra CSV columns are ignored by the schema.
  • For allowed_values, the agent will attempt to correct close matches (typos, casing) before dropping.
  • Rows with unfixable violations in required fields are dropped — the agent won't invent data.
  • Start simple: define just dtype and nullable first, then add more rules.
